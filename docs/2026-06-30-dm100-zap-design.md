# DM-100 Electricity Zap — Design Spec

## Summary

Add DM-100's ranged electric zap attack. Reuses the existing `LIGHTNING_ARC` event pipeline (cyan arc + sparks + LIGHTNING sound). Backend only — no frontend changes required.

## Context

- DM-100 (`mobs.py:526`) has `attack_range: 8` and `properties: ["ELECTRIC", "INORGANIC"]` but no custom AI mixin.
- It currently falls through to the default melee path in `tick.py`.
- The `LIGHTNING_ARC` frontend handler (`combat.ts:321`) already renders a cyan lightning arc, spawns spark particles, and plays `LIGHTNING` sound.
- Pattern reference: `ai_dm200.py` / `DM200AIMixin`.

## New File: `backend/app/engine/game/ai_dm100.py`

```python
class DM100AIMixin:
    """DM-100: fires electric zap at range with cooldown."""

    def _update_dm100(self, mob, floor, floor_id: int) -> bool:
        target = self._find_nearest_player(mob.pos, floor_id)
        if target is None or mob.ai_state != "hunting":
            return False

        dist = self._get_distance(mob.pos, target.pos)
        in_los = self._is_in_los(mob.pos, target.pos, floor_id=floor_id)

        if dist == 1:
            return False  # let default melee handle adjacency

        zap_cooldown = getattr(mob, "zap_cooldown", 0)
        if zap_cooldown > 0:
            mob.zap_cooldown = max(0, zap_cooldown - 1)

        if dist <= mob.attack_range and in_los and zap_cooldown <= 0:
            self._dm100_zap(mob, target, floor_id)
            return True

        return False

    def _dm100_zap(self, mob, target, floor_id: int):
        self.add_event("LIGHTNING_ARC", {
            "source_x": mob.pos.x, "source_y": mob.pos.y,
            "target_x": target.pos.x, "target_y": target.pos.y,
        }, floor_id=floor_id)
        self.add_event("PLAY_SOUND", {"sound": "LIGHTNING"}, floor_id=floor_id)

        mob.last_attack_time = time.time()
        mob.zap_cooldown = 3

        dmg = random.randint(mob.damage_min, mob.damage_max)
        dr = random.randint(target.get_dr_min(), target.get_dr_max())
        dealt = target.take_damage(max(0, dmg - dr))

        self.add_event("ATTACK", {
            "source": mob.id, "target": target.id,
            "damage": dealt, "surprise": False,
        }, floor_id=floor_id)
        self.add_event("DAMAGE", {
            "target": target.id, "amount": dealt, "shock": True,
        }, floor_id=floor_id)

        if not target.is_alive:
            self.add_event("DEATH", {"target": target.id}, floor_id=floor_id)
```

## Changes to `tick.py`

1. Import `DM100AIMixin` alongside other mixin imports.
2. Import `DM100` alongside other mob class imports.
3. In the mob update loop (after `_update_dm200` block ~line 446):

```python
if isinstance(mob, DM100):
    if self._update_dm100(mob, floor, floor_id):
        continue
```

## What's NOT changing

- Frontend `combat.ts` — `LIGHTNING_ARC` handler already covers visuals and sound.
- Event schemas — `LIGHTNING_ARC` schema unchanged.
- DM-100 entity definition in `mobs.py` — no new fields needed (`zap_cooldown` is a runtime attribute set dynamically like `vent_cooldown` on DM-200).

## Decisions

| Decision | Choice | Reason |
|---|---|---|
| Cooldown | 3 ticks | Prison-floor regular enemy; ~2-3 turn cadence matches SPD feel |
| Skip at dist==1 | Yes | Default melee handles adjacency, same as DM-200 |
| Chain lightning | No | DM-100 is not a wand; plain hit only |
| Shock buff | No | Keeping simple; buff is a wand mechanic |
| Visual | Reuse LIGHTNING_ARC | Zero frontend work; cyan arc + sparks + sound already match |
